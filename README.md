# Taller 02: Modelo de Información (ERD + Diagrama de Contexto)

## Objetivo
Modelar el **modelo de información** de un proceso mediante:
1) Un **ERD (Modelo Entidad-Relación)** con entidades, relaciones y cardinalidades.
2) Un **diagrama de contexto de negocio**, mostrando actores, sistemas y flujos de datos.

## Caso base (clase): Clínica Salud Viva
En clase se trabajó con el proceso de **Agendamiento de Citas Médicas** para construir:
- Un ERD simple con entidades principales: Paciente, Cita, Médico, Especialidad, Factura.
- Un borrador del diagrama de contexto (actores + sistemas + datos que fluyen).

## Cliente real (entrega): Heladería – Pedido de Helado de Yogur
Después de clase se adaptó el modelo al dominio real del cliente:
- ERD ajustado al proceso de pedido y personalización (sabores, tamaño, toppings, salsas).
- Diagrama de contexto con POS como núcleo e integración con inventario y pagos.

## Herramientas
- Lucidchart
- Notación ERD (crow’s foot) y diagrama de contexto
